# Guardian Incident Response Runbook

## Purpose

Defines the operator process for responding to Guardian security incidents.

## Steps

1. Confirm incident alert.
2. Identify guild and severity.
3. Confirm actor and target where available.
4. Check containment status.
5. Check punishment status.
6. Check recovery status.
7. Review forensic logs.
8. Record final verdict.

## Stop Condition

Escalate immediately if Guardian loses permissions, is kicked, or cannot log evidence.
