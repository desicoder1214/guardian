# Guardian Implementation Roadmap

## 1. Purpose

This document defines the approved implementation roadmap for Guardian.

The roadmap prevents rushed implementation, architecture drift, incomplete security controls, and uncertified releases.

---

## 2. Global Rules

- Implementation must follow documentation order.
- Security Kernel work takes priority over optional modules.
- No feature may be implemented without traceability.
- No module may be considered complete without tests.
- No release may be certified without evidence.
- Optional modules must never block containment, punishment, recovery, or forensics.
