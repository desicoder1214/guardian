# Guardian Change Management Contract

## 1. Purpose

This document defines Guardian's change management process.

Every change must be documented, reviewed, risk assessed, tested, and approved before becoming part of the production architecture.

---

## 2. Global Rules

- No undocumented change is permitted.
- Every change must reference affected contracts.
- Security-critical changes require architecture review.
- Breaking changes require migration and rollback plans.
- All changes must be traceable through version history.
- Certification impact must be documented.
