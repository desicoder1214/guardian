# Guardian Production Readiness Gate

## 1. Purpose

This document defines the final production-readiness gate for Guardian.

Guardian is production-ready only when architecture, contracts, implementation, tests, runbooks, evidence, and certification are complete.

---

## 2. Global Rules

- Production readiness must be evidence-based.
- Security-critical systems must pass certification gates.
- Missing logs or forensics block production readiness.
- Failed live drills block production readiness.
- Rollback must be documented before release.
- Operator runbooks must be complete.
- Executive approval is required for production launch.

---
## Area: Production Readiness Philosophy

### Purpose
Defines production-readiness requirements for `PRODUCTION_READINESS_PHILOSOPHY`.

### Required Evidence
- Documentation evidence
- Contract evidence

- Risk review
- Approval status

### Readiness Rules
- Must satisfy documented criteria.
- Must not have unresolved critical blockers.
- Must have rollback strategy.
- Must be auditable.

### Blocking Conditions
- Failed certification gate
- Failed critical test
- Missing forensic evidence
- Missing rollback plan
- Unresolved security risk
