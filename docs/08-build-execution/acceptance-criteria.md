# Guardian Acceptance Criteria

## 1. Purpose

This document defines the acceptance criteria required before Guardian work can be considered complete.

A feature is not accepted because it was implemented. It is accepted only when architecture, contracts, tests, evidence, and certification criteria are satisfied.

---

## 2. Global Rules

- Every feature must have documented acceptance criteria.
- Security features require evidence-based acceptance.
- Acceptance must include tests and logs.
- Critical security failures block acceptance.
- Optional modules cannot be accepted if they weaken the Security Kernel.
- Acceptance decisions must be auditable.

- Must preserve security boundaries.
- Must produce required logs.
- Must not introduce unresolved blockers.

### Rejection Conditions
- Missing tests
- Missing evidence
- Failed security behavior
- Architecture drift
- Unresolved critical defect
