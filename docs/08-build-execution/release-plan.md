# Guardian Release Plan

## 1. Purpose

This document defines Guardian's release planning process.

A Guardian release is not approved because code exists. A release is approved only when scope, tests, evidence, rollback, monitoring, and certification are complete.

---

## 2. Global Rules

- Every release must have documented scope.
- Every release must map to contracts and tests.
- Security-critical releases require regression evidence.
- Live-drill evidence is required where destructive behavior is involved.
- Rollback must be defined before release.
- Release approval must be auditable.

- Critical defect requires rollback decision.
- Missing evidence blocks certification.
